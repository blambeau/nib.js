Do exports.XXX and require('./XXX') look painful?

Sometimes, you want to maintain your code source "as if it were in a single file" but keeping the ability to split different features in different files... In that case, you would like to avoid having to exports and require everything everytime.

This should only be used if your code 1) has no name clash accross different files and 2) do not expect a particular load order between the different files.
